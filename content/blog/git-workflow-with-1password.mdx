# 用 1Password SSH Agent 管理多身份 Git 工作流

同时维护公司项目和个人项目时，最常见的痛点：SSH 密钥混用、Git 提交身份串号、每次 pull/push 都要手动输入密码短语。本文将 1Password SSH Agent + SSH Host 别名 + Git 条件包含三管齐下，彻底解决这些问题。

## 当前环境的问题

先看看一个典型的多项目开发环境：

```bash
# ~/.ssh/ 下只有一个 RSA 密钥
$ ls ~/.ssh/
id_rsa  id_rsa.pub

# Git 全局用固定身份
$ git config --global user.name
test-wangyinggang
$ git config --global user.email
kingyinggang@gmail.com
```

这意味着什么？

- **公司和个人仓库用同一把 SSH 密钥** — 万一私钥泄露，两个 GitHub 账号同时沦陷。
- **所有 commit 都用同一个邮箱** — 在公司仓库里暴露个人 Gmail，反之亦然。
- **RSA 密钥不如 Ed25519** — 更长的密钥长度、更慢的签名速度。
- **没有 SSH Agent 管理** — 每次操作可能都要手动 `ssh-add`，繁琐且不安全。

## 目标架构

```
┌─────────────────────────────────────┐
│            1Password                │
│  ┌───────────┐  ┌───────────┐     │
│  │ Personal   │  │ Work      │     │
│  │ SSH Key    │  │ SSH Key   │     │
│  │ (ed25519)  │  │ (ed25519) │     │
│  └───────────┘  └───────────┘     │
│         │              │           │
│         └──────┬───────┘           │
│         ┌──────▼──────┐            │
│         │  SSH Agent  │            │
│         └──────┬──────┘            │
└────────────────┼───────────────────┘
                 │ SSH_AUTH_SOCK
    ┌────────────▼────────────┐
    │     ~/.ssh/config       │
    │  Host personal → key A  │
    │  Host work     → key B  │
    └────────────┬────────────┘
                 │
    ┌────────────▼────────────┐
    │    Git Remote (别名)     │
    │  github.com-personal    │
    │  github.com-work        │
    └─────────────────────────┘

    ┌──────────────────────────┐
    │     ~/.gitconfig         │
    │  includeIf "gitdir:      │
    │    ~/work/ → 公司身份    │
    │    ~/personal/ → 个人身份│
    └──────────────────────────┘
```

## 第一步：在 1Password 中创建 Ed25519 密钥

打开 1Password 桌面 App，新建两个 SSH Key 项：

1. **Personal SSH Key (GitHub)** — 密钥类型选择 Ed25519，点击生成
2. **Work SSH Key (GitHub)** — 同样生成 Ed25519

保存后分别复制公钥，后续需要上传到对应的 GitHub 账户 Settings → SSH and GPG keys 中。

> Ed25519 比 RSA 更安全、签名更快、密钥更短。GitHub 和 GitLab 均已完全支持。

## 第二步：启用 1Password SSH Agent

打开 1Password → 设置 → 开发者：

- 勾选 **"使用 SSH agent"**
- 在密钥列表中，勾选刚创建的两个 SSH Key

在 `~/.zshrc` 中添加强制指向 1Password Agent：

```bash
export SSH_AUTH_SOCK=~/Library/Group\ Containers/2BUA8C4S2C.com.1password/t/agent.sock
```

新开终端后验证：

```bash
$ echo $SSH_AUTH_SOCK
/Users/you/Library/Group Containers/2BUA8C4S2C.com.1password/t/agent.sock

$ ssh-add -l
256 SHA256:xxxx Personal SSH Key (GitHub) (ED25519)
256 SHA256:yyyy Work SSH Key (GitHub) (ED25519)
```

这一步的核心价值：**私钥文件永远不会出现在磁盘上，1Password 只在授权时才通过 Agent 临时使用它。** 换机器时，密钥通过 1Password 同步，无需手动搬运 `~/.ssh/`。

## 第三步：配置 SSH Host 别名

编辑 `~/.ssh/config`：

```ssh-config
# 个人 GitHub
Host github.com-personal
  HostName github.com
  User git
  IdentitiesOnly yes

# 公司 GitHub
Host github.com-work
  HostName github.com
  User git
  IdentitiesOnly yes

# 默认规则
Host *
  AddKeysToAgent yes
  UseKeychain yes
  IdentityAgent ~/Library/Group Containers/2BUA8C4S2C.com.1password/t/agent.sock
```

**不写 `IdentityFile`**，因为密钥在 1Password Agent 里，SSH 会自动通过 Agent 找到匹配的密钥。`IdentitiesOnly yes` 确保只尝试该 Host 对应的那个密钥，避免 Agent 一个个试过去。

### 对应的远程仓库地址

| 场景 | 旧地址 | 新地址 |
|------|--------|--------|
| 个人项目 | `git@github.com:you/project.git` | `git@github.com-personal:you/project.git` |
| 公司项目 | `git@github.com:company/repo.git` | `git@github.com-work:company/repo.git` |

修改已有仓库的 remote：

```bash
# 个人仓库中
git remote set-url origin git@github.com-personal:you/project.git

# 公司仓库中
git remote set-url origin git@github.com-work:company/repo.git
```

验证连接：

```bash
ssh -T git@github.com-personal
# Hi you! You've successfully authenticated...

ssh -T git@github.com-work
# Hi company-user! You've successfully authenticated...
```

## 第四步：按目录自动切换 Git 提交身份

这是最容易踩坑的地方。当你 `cd` 到公司项目后，`git commit` 默认用的是全局 `user.name` 和 `user.email`，不小心就会在公司 repo 里留个人邮箱。

Git 的 `includeIf` 可以按目录自动切换身份：

编辑 `~/.gitconfig`：

```ini
[user]
  # 兜底默认值
  name = Your Default Name
  email = your-default@email.com

[includeIf "gitdir:~/work/"]
  path = ~/.gitconfig-work

[includeIf "gitdir:~/personal/"]
  path = ~/.gitconfig-personal
```

创建 `~/.gitconfig-work`：

```ini
[user]
  name = Yinggang Wang
  email = yinggang@company.com
```

创建 `~/.gitconfig-personal`：

```ini
[user]
  name = wangyinggang
  email = kingyinggang@gmail.com
```

**效果**：只要仓库目录在 `~/work/` 下，自动使用公司身份；在 `~/personal/` 下，自动使用个人身份。完全无需手动 `git config user.name ...`。

## 第五步：清理旧密钥

上传完新公钥并验证连接后，就可以删掉旧的了：

```bash
# 备份并删除旧 RSA 密钥
rm ~/.ssh/id_rsa ~/.ssh/id_rsa.pub

# 在 GitHub 设置中删除旧公钥
# Settings → SSH and GPG keys → 找到旧的 RSA key → Delete
```

## 完整验证

```bash
# 1. 确认 Agent 中有两把密钥
ssh-add -l

# 2. 确认连接
ssh -T git@github.com-personal
ssh -T git@github.com-work

# 3. 在个人项目中测试 commit 身份
cd ~/personal/my-project
git config user.email  # 应该输出个人邮箱
git commit --allow-empty -m "test identity"

# 4. 在公司项目中测试
cd ~/work/company-repo
git config user.email  # 应该输出公司邮箱
```

## 总结

这套方案的核心思路：

- **密钥安全**：1Password 托管 ed25519 私钥，永不离保险库
- **连接无感**：SSH Agent 自动提供密钥，无需每次 `ssh-add`
- **身份隔离**：Host 别名 → 不同密钥，目录规则 → 不同邮箱
- **跨设备同步**：换电脑只需装 1Password + 配一次 `~/.ssh/config` 和 `~/.gitconfig`
